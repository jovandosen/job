2. Koja je razlika izmedju varijable i konstante ?

-> varijabla

	-> vrednost varijable se moze menjati u toku izvrsavanja skripte
	-> naziv varijable pocinje sa dolar ($) znakom
	-> naziv varijable ne sme poceti sa brojem, karakter definisan posle dolar ($) znaka
		-> $123 = 123 nije ok 
		-> $_123 = 123 je ok

-> konstanta

	-> vrednost konstante se ne moze menjati u toku izvrsavanja skripte
	-> naziv konstante ne sme da pocne sa dolar ($) znakom
	-> naziv konstante moze da pocne sa donjom crtom (_)
	-> konstanta se definise pomocu define() funkcije, npr. define('TEST', 100);
	-> scope konstante je global, sto bi znacilo da je dostupna svuda

3. Koji scope-ovi (dostupnost) varijable postoje ?

-> global

	<?php
		$x = 10;

		function test()
		{
			// $x nije dostupan u funkciji
			// $x moze biti dostupan u funkciji ako dodamo kljucnu rec global ispred $x
			// global $x
		}

		// $x je dostupan van funkcije
	?>

	-> PHP smesta globalne varijable u asocijativni niz $GLOBALS[index], npr. $GLOBALS['x']

-> local

	<?php
		function foo()
		{
			$y = 5;
			// $y ima local scope i dostupan je samo unutar funkcije
		}
	?>

-> static

	-> Kada se skripta izvrsi, sve varijable se unistavaju
	-> Ako zelimo da sacuvamo vrednost neke varijable mozemo da dodamo kljucnu rec static

	<?php
		function myTest() 
		{
		  	static $x = 0;
		  	echo $x;
		  	$x++;
		}

		myTest();
		myTest();
		myTest();
	?>

4. Sta je session ?

-> Sesija (session) je nacin skladistenja informacija u promenljive tj. varijable.
-> Te varijable su dostupne kod vise strana (across multiple pages)
-> Sesije se ne cuvaju na racunaru korisnika
-> Sesije traju sve dok korisnik ne zatvori pretrazivac (browser)
-> HTTP adrese ne zatrzavaju stanje, sto je reseno pomocu sesija, kako bi neki podatak postojao kroz vise HTTP adresa tj. strana
-> Sesije se pokrecu pomocu funkcije session_start(), ova funkcija mora biti pozvana na pocetku svakog php fajla gde zelimo da koristimo session varijable
-> Funkcija session_start() se poziva pre bilo kog HTML koda, na samom vrhu fajla
-> Sve session varijable se nalaze u globalnoj varijabli tj. nizu $_SESSION

-> Primer: index.php

	<?php
	// Start the session
	session_start();
	?>
	<!DOCTYPE html>
	<html>
	    <body>

	        <?php
	            // Set session variables
	            $_SESSION["favcolor"] = "green";
	            $_SESSION["favanimal"] = "cat";
	            $_SESSION["test"] = "testing";
	            echo "Session variables are set.";
	        ?>

	    </body>
	</html>

-> Primer 2: foo.php

	<?php
	session_start();
	?>
	<!DOCTYPE html>
	<html>
		<body>

		<?php
			// Echo session variables that were set on previous page
			echo "Favorite color is " . $_SESSION["favcolor"] . ".<br>";
			echo "Favorite animal is " . $_SESSION["favanimal"] . ".";
		?>

		</body>
	</html>

-> Ako zelimo da promenimo vrednost sesije, samo dodelimo novu vrednost

	<?php
	session_start();
	?>
	<!DOCTYPE html>
	<html>
		<body>

		<?php
			// to change a session variable, just overwrite it
			$_SESSION["favcolor"] = "yellow";
			print_r($_SESSION);
		?>

		</body>
	</html>

-> Ako zelimo da uklonimo sve session varijable i unistimo sesiju, to mozemo pomocu funkcija: session_unset() i session_destroy()

	<?php
	session_start();
	?>
	<!DOCTYPE html>
	<html>
		<body>

		<?php
			// remove all session variables
			session_unset();

			// destroy the session
			session_destroy();
		?>

		</body>
	</html>

-> Sesije se cuvaju na serveru
-> u php.ini fajlu je definisana lokacija session.save_path
-> U mom slucaju, na linux masini, session.save_path je /var/lib/php/sessions
-> Pristup /var/lib/php/sessions direktorijumu je strogo zabranjen
-> Sesija se cuva u vidu user-key
	-> npr. 765487cf34ert8dede5a562e4f3a7e12

7. Da li je php case-sensitive ?

-> PHP je delimicno case-sensitive
-> ugradjene (build-in) funkcije i custom funkcije nisu case-sensitive
-> nazivi varijabli su case-sensitive

-> case sensitive
	-> variables, constants, array keys, class properties, class constants

-> case insensitive
	-> functions, class constructors, class methods, keywords and constructs (if, else, null, foreach, echo etc.)

9. Koja su pravila prilikom deklarisanja varijable ?

-> varijabla mora poceti sa dolar znakom
-> ne sme poceti brojem
-> moze da se sastoji iz slova i brojeva
-> moze poceti sa donjom crtom
-> ne sme da sadrzi specijane karaktere kao sto su: +, -, /, *, %, ^, &
-> ne sme da sadrzi prazna mesta (space)
-> $name i $NAME su razlicite promenljive, sto bi znacilo da su promenljive case-sensitive

15. Koji tipovi gresaka postoje ?

-> Notice

	-> nece zaustaviti izvrsavanje skripte
	-> nisu vidljivi korisniku
	-> npr. promenljiva koja nije definisana (undefined variable)

-> Warning

	-> nece zaustaviti izvrsavanje skripte
	-> vidljivi su korisniku
	-> malo su ozbiljniji od Notice-a
	-> npr. pokusavamo da include-ujemo skriptu (fajl) koja ne postoji

-> Fatal

	-> Odmah prekidaju izvrsavanje skripte

	-> npr. ako zelimo da require-ujemo fajl koji ne postoji

16. Sta je trait ?

-> PHP ne podrzava nasledjivanje vise klasa u isto vreme tj. jedna klasa moze da nasledi samo jednu klasu
-> Trait-ovi resavaju ovaj problem, tako da mozemo da koristimo razlicite funkcije (definisane unutar trait-a) u nekoj klasi
-> Trait se ne moze instancirati
-> Funkcije definisane unutar trait-a moraju da imaju vidljivost public

-> Primer:

<?php

	trait Test
	{
		public function test()
		{
			echo 'all good';
		}
	}

	class Foo
	{
		use Test;
	}

	$foo = new Foo();

	$foo->test();

	?>

19. Sta je cookie ?

-> Cookie se koristi za skladistenje informacija
-> Cookie se cuva na racunaru korisnika tj. u pretrazivacu (browser)
-> Mora biti definisan pre bilo kog HTML koda
-> Cookie se kreira pomocu funkcije setcookie(), ista funkcija se koristi za brisanje cookie-ja
-> Samo je naziv cookie-ja obavezan parametar, svi drugi parametri su opcioni
-> Svi kreirani cookie-ji se nalaze u nizu $_COOKIE
-> Ako je vrednost session.cookie_secure postavljena na 1 (u php.ini fajlu), samo HTTPS web sajtovi mogu da koriste cookie-je 

-> Primeri: 

	-> Primer 1:

		<?php
			$value = 'something from somewhere';

			setcookie("TestCookie", $value);
			setcookie("TestCookie", $value, time()+3600);  /* expire in 1 hour */
			setcookie("TestCookie", $value, time()+3600, "/~rasmus/", "example.com", 1);
		?>

	-> Primer 2 (brisanje cookie-ja):

		<?php
			// set the expiration date to one hour ago
			setcookie("TestCookie", "", time() - 3600);
			setcookie("TestCookie", "", time() - 3600, "/~rasmus/", "example.com", 1);
		?>

	-> Primer 3 (cookie kao niz):

		<?php
			// set the cookies
			setcookie("cookie[three]", "cookiethree");
			setcookie("cookie[two]", "cookietwo");
			setcookie("cookie[one]", "cookieone");

			// after the page reloads, print them out
			if (isset($_COOKIE['cookie'])) {
			    foreach ($_COOKIE['cookie'] as $name => $value) {
			        $name = htmlspecialchars($name);
			        $value = htmlspecialchars($value);
			        echo "$name : $value <br />\n";
			    }
			}
		?>

22. Sta je Path Traversal ?

-> Path Traversal je vrsta hakerskog napada (zove se jos i dot-dot-slash attack)
-> Npr. ako haker zeli da pristupi osetljivim podacima preko URL-a, koriscenjem ../ 
-> ../ se koristi za kretanje kroz direktorijume (jedan nivo gore)
-> Na taj nacin haker moze da dodje do direktorijuma koji su van root-a ( /etc/passwd )
