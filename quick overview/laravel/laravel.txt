1. Laravel 9.x zahteva minimum verziju PHP 8.0

2. Umesto webpack se koristi vite

	-> vite se koristi za kompajliranje CSS i JS koda

3. Instalacija novog projekta putem composer-a

	-> composer create-project laravel/laravel quick-overview-app

4. .env 

	-> .env je glavni konfiguracioni fajl
	-> u njemu definisemo parametre za bazu, za slanje email-ova (smtp data), secret id i secret key od drustvenih mreza 
		(ako zelimo druge login opcije putem: gmail, linkedin, github, bitbucket)

5. @csrf

	-> cross site request forgery
	-> kreiranje zastitnog tokena u slucaju hakerskog napada
	-> kreirani token se smesta u sesiju i u formu u vidu hidden input polja

6. composer

	-> alata za instaliranje iskljucivo php biblioteka koda
	-> composer.json je glavni konfiguracioni fajl
	-> composer.lock je fajl sa meta podacima (podaci o instaliranim bibliotekama)

7. npm

	-> Node Package Manager
	-> sluzi za instaliranje biblioteka koda (css i js)
	-> neophodan je Node.js (open source server environment)
	-> package.json je glavni konfiguracioni fajl
	-> npm install ce kreirati direktorijum (node_modules) sa svim bibliotekama koda

8. homebrew (brew)

	-> slican composer-u
	-> sluzi za instaliranje biblioteka koda
	-> mac ili lunux (radno okruzenje)

9. Struktura direktorijuma (najvazniji direktorijumi)

	-> app
	-> bootstrap
	-> config
	-> database
	-> public
	-> lang
	-> resources
	-> routes
	-> storage
	-> tests
	-> vendor

10. app direktorijum

	-> sastoji se iz: 

		-> Controllers
		-> Models
		-> Middleware
		-> Providers
		-> Custom created artisan commands

11. bootstrap direktorijum

	-> sastoji se iz app.php fajla, koji prakricno pokrece celu laravel aplikaciju

12. config direktorijum

	-> sastoji se iz konfiguracionih fajlova

		-> app.php (APP_NAME, APP_ENV, APP_DEBUG, timezone)

		-> database.php 

			-> drajveri:

				-> mysql
				-> sqlite
				-> pqsql
				-> sqlsrv

		-> filesystems.php (gde se smestaju upload-ovani fajlovi, kreirani fajlovi i sl.) 

			-> local storage path
			-> s3

		->  mail.php (drajveri)

			-> smtp
			-> mailgun
			-> postmark

		-> services.php (osetljivi podaci -> secret id i secret key)

			-> gmail
			-> github
			-> bitbucket
			-> paypal
			-> stripe

13. database direktorijum

	-> sastoji se iz migracija
	-> Factories (kreiranje dummy podataka)
	-> db seeds (kreiranje dummy podataka)

14. public direktorijum

	-> ulazna putanja u aplikaciju (sluzi kao front controller)
	-> sastoji se iz:

		-> index.php fajla koji je front controller
		-> .htaccess fajla u kom su definisane direktive (svi HTTP zahtevi da se redirektuju na index.php)
		-> robots.txt (za indeksiranje)

15. lang direktorijum

	-> ukoliko je aplikacija visejezicna
	-> tekst, validacione poruke i sl.

16. resources direktorijum

	-> svi blade template fajlovi se nalaze u ovom direktorijumu
	-> partials (header. footer, sidebar)
	-> main.blade.php

17. routes direktorijum 

	-> u ovom direktorijumu se nalaze sve rute (GET, POST, PATCH, DELETE) aplikacije

18. storage direktorijum

	-> sastoji se iz log fajlova, upload-ovanih fajlova i sl.

19. tests direktorijum 

	-> svi testovi se nalaze u ovom direktorijumu, npr. PHPUnit tests

20. vendor direktorijum

	-> sve php biblioteke koje smo instalirali putem composer-a se nalaze u ovom direktorijumu

21. Breeze

	-> fast auth system (register, login, password change, mail confirmation)
	-> podrzava Vue i React

22. Rutiranje

	-> vrste ruta:

		Route::get($uri, $callback);
		Route::post($uri, $callback);
		Route::put($uri, $callback);
		Route::patch($uri, $callback);
		Route::delete($uri, $callback);
		Route::options($uri, $callback);

	-> dodeljivanje imena rutama:

		Route::get('/user/profile', function () {
    		//
		})->name('profile');

	-> pozivanje middleware-a na grupu ruta:

		Route::middleware(['first', 'second'])->group(function () {
		    Route::get('/', function () {
		        // Uses first & second middleware...
		    });
		 
		    Route::get('/user/profile', function () {
		        // Uses first & second middleware...
		    });
		});

	-> prosledjivanje parametara u rute:
	
		Route::get('/user/{id}', function ($id) {
    		return 'User '.$id;
		});

23. Validacija formi

	->primer:

		$validated = $request->validate([
        	'title' => 'required|unique:posts|max:255',
        	'body' => 'required',
    	]);
